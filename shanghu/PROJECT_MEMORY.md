## 关键业务页
### 游戏控制
`game-control.html` 是当前最重要的功能页，包含：
- 平台 ID 输入
- 按游戏品牌 / 按游戏调控
- 控输 / 控赢
- RTP 档位和快捷值
- 解除类型：金额 / RTP / 时间
- 表单校验与保存提示

页面里的核心逻辑都写在同一个 Vue `setup()` 里，状态变量包括 `form`、`rules`、`rtpGearValue`、`releaseAmountValue` 等。

### API 接入配置
`api-access.html` 是当前浏览器打开页，页面结构包括：
- API 对接配置：API 模式、商户 ID、秘钥、对接文档、玩家信息/余额/改余额回调地址、返回大厅跳转地址。
- API 域名地址参考：用 `details/summary` 分组展示公共、单一钱包、转账钱包、报表相关、功能相关接口。
- 签名算法校验工具：本页内置 MD5 实现，支持输入 body JSON 字符串并输出签名。
- 页面里有复制按钮逻辑，点击 `data-copy` 元素会写入剪贴板并短暂显示“已复制”。

### 报表与数据页
`game-report.html`、`platform-profit.html`、`daily-report.html` 已经转向“筛选栏 + 导出按钮 + 横向数据表 + 底部分页”的数据界面。
`merchant-report.html` 当前是按月筛选的商户报表，不展示商户筛选、商户列和顶部汇总卡片，表格数据居中，表格右上角只保留导出图标。

## 已确认的项目需求偏好
- 页面类型：静态管理后台原型，优先 PC 端展示，同时兼顾常见分辨率适配。
- 导航体验：左侧菜单和顶部导航要稳定、清晰，视觉上不要“方块生硬感”。
- 左侧菜单：
  - 所有页面必须通过 `sidebar.js` 引入统一菜单，菜单 HTML、图标、active 高亮、展开/收起逻辑只允许写在这一份公共组件里。
  - 页面内禁止单独写菜单 HTML / 菜单 CSS / 菜单 JS，避免样式分叉、重复绑定和点击闪烁。
  - 当前页面高亮必须由 `sidebar.js` 基于路由同步生成，不能依赖页面加载后的二次 JS 修正。
  - 菜单风格参考白底后台样式，不要深色侧栏。
  - 图标要偏线性、现代，选中态用浅灰底 + 蓝色文字/图标。
  - 顶部汉堡按钮需要真实支持侧栏收起/展开。
  - 凡是有二级菜单的一级菜单，只能负责展开/收起，不能跳转到一级数据页面；点击一级菜单整行也只做展开/收起。
  - 子菜单箭头需要能控制对应子项展开/收起。
  - 任何弹出层都不能压住或盖住左侧菜单栏。
- 游戏选择器：
  - 优先图标快速选择，支持分页（可承载 100 款以内游戏）。
  - 支持模糊搜索，且可按中文名、英文名、游戏 ID 检索。
  - 默认状态不选具体游戏时即视为“全部游戏”。
  - 游戏选择弹出框必须适配当前视口，位置要避开侧栏，不能向左溢出覆盖菜单。
- 筛选能力：
  - 平台 ID 查询必须真实生效。
  - 日期需要支持起止区间（从某日到某日），标签文案用“日期选择”，不要用“创建时间”。
  - 日期选择控件要紧凑，不能被“查询 / 重置”压住。
  - 在常见 PC 宽度下，平台 ID、游戏选择、日期选择、查询、重置尽量保持同一行；如果空间不足再换行。
  - 查询、重置按钮需要保持在同一行，行为要符合直觉且有实际数据变化。
- 表格与数据：
  - 数据对齐和排版要工整（居中展示、错位少）。
  - 玩家标签需要业务分层：低端玩家 / 中端玩家 / 优质玩家 / 疑似刷分。
  - 平台 ID 复制按钮必须可用，且图标视觉要更现代。
- 分页区：
  - 需要每页条数选择器（50 / 100 / 500 / 1000），默认 100。
- 报表 / 数据页：
  - 游戏数据、平台盈亏、每日报告这类页面优先做成“筛选栏 + 导出按钮 + 横向数据表 + 底部分页”的后台数据界面。
  - 商户报表也按同类数据页处理：月份筛选、重置/搜索、快捷月份、导出入口、表格和分页都要齐全。
  - 商户报表不展示商户筛选和表格商户列；筛选区只按月份筛选，不能选日范围；表格日期只显示月份，例如 `2026-04`。
  - 商户报表不展示顶部汇总指标卡片，明细表作为主要数据展示。
  - 商户报表表头不放问号提示点；表格数据居中展示，金额保留 2 位小数和 tabular 数字，表格右上角只保留导出图标并带 hover 提示。
  - 不要再用大号摘要卡片作为主要数据展示，截图里是明细表优先。
  - 游戏数据、平台盈亏、每日报告这类明细页可保留问号提示点、游戏缩略图、金额强调色、单行数据和分页控件；商户报表按最新反馈不显示问号提示点。
  - 页面命名上，“游戏报告”应逐步统一为“游戏数据”，面包屑和左侧菜单也要同步。
