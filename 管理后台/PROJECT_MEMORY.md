## 已确认的项目需求偏好
- 页面类型：静态管理后台原型，优先 PC 端展示，同时兼顾常见分辨率适配。
- 导航体验：左侧菜单和顶部导航要稳定、清晰，视觉上不要“方块生硬感”。
- 左侧菜单：
  - 所有页面必须通过 `sidebar.js` 引入统一菜单，菜单 HTML、图标、active 高亮、展开/收起逻辑只允许写在这一份公共组件里。
  - 页面内禁止单独写菜单 HTML / 菜单 CSS / 菜单 JS，避免样式分叉、重复绑定和点击闪烁。
  - 当前页面高亮必须由 `sidebar.js` 基于路由同步生成，不能依赖页面加载后的二次 JS 修正。
  - 菜单风格参考白底后台样式，不要深色侧栏。
  - 图标要偏线性、现代，选中态用浅灰底 + 蓝色文字/图标。
  - 顶部汉堡按钮需要真实支持侧栏收起/展开。
  - 凡是有二级菜单的一级菜单，只能负责展开/收起，不能跳转到一级数据页面；点击一级菜单整行也只做展开/收起。
  - 子菜单箭头需要能控制对应子项展开/收起。
  - 任何弹出层都不能压住或盖住左侧菜单栏。
- 游戏选择器：
  - 优先图标快速选择，支持分页（可承载 100 款以内游戏）。
  - 支持模糊搜索，且可按中文名、英文名、游戏 ID 检索。
  - 默认状态不选具体游戏时即视为“全部游戏”。
  - 游戏选择弹出框必须适配当前视口，位置要避开侧栏，不能向左溢出覆盖菜单。
  - 游戏选择弹出框搜索框上方需要支持品牌筛选，常用品牌包括 `PG`、`PP`、`JILI`、`SPRLBE`；每个品牌可承载上百款游戏，品牌筛选和搜索需要叠加生效。
- 筛选能力：
  - 平台 ID 查询必须真实生效。
  - 日期需要支持起止区间（从某日到某日），标签文案用“日期选择”，不要用“创建时间”。
  - 日期选择控件要紧凑，不能被“查询 / 重置”压住。
  - 在常见 PC 宽度下，平台 ID、游戏选择、日期选择、查询、重置尽量保持同一行；如果空间不足再换行。
  - 查询、重置按钮需要保持在同一行，行为要符合直觉且有实际数据变化。
- 表格与数据：
  - 数据对齐和排版要工整（居中展示、错位少）。
  - 玩家标签需要业务分层：低端玩家 / 中端玩家 / 优质玩家 / 疑似刷分。
  - 平台 ID 复制按钮必须可用，且图标视觉要更现代。
- 分页区：
  - 需要每页条数选择器（50 / 100 / 500 / 1000），默认 100。
- 报表 / 数据页：
  - 游戏数据、平台盈亏、每日报告这类页面优先做成“筛选栏 + 导出按钮 + 横向数据表 + 底部分页”的后台数据界面。

  - 商户报表不展示顶部汇总指标卡片，明细表作为主要数据展示。
  - 商户报表表头不放问号提示点；表格数据居中展示，金额保留 2 位小数和 tabular 数字，表格右上角只保留导出图标并带 hover 提示。
  - 不要再用大号摘要卡片作为主要数据展示，截图里是明细表优先。
  - 游戏数据、平台盈亏、每日报告这类明细页可保留问号提示点、游戏缩略图、金额强调色、单行数据和分页控件；商户报表按最新反馈不显示问号提示点。

- 表格列宽规则：表格外层铺满后，行内列不能继续只用固定像素宽度；要使用 `minmax(最小宽度, fr)` 这类响应式列宽，让宽屏时每一列按比例拉伸填满整行，窄屏时再出现横向滚动。
- 表格容器规则：横向表格应 `width: 100%` + 合理 `min-width`，不能因为 `min-width` 小于容器宽度而把空白堆到最右侧。
- 分页规则：数据页分页必须在卡片底部，使用 `margin-top: auto` 或等价布局沉底；数据少的时候也不能贴在表格数据行下面。
- 横向滚动条规则：所有数据表横向滚动条都要统一成细轨道、圆角滑块、hover 变深的样式，不要使用浏览器默认粗糙横杠。
- 游戏控制页规则：表单卡片也要填满 PC 宽屏；textarea 和主要输入区不要被小的 `max-width` 限制，避免右侧大面积空白。底部操作按钮保持在底部区域，适合演示和操作。
- 新建页面检查清单：完成新页面后必须主动检查 PC 宽屏下是否铺满、表格列是否拉伸、分页是否沉底、筛选栏是否同一行优先、控件是否不重叠、横向滚动条是否好看。
- 2026-05-08 补充：配置类页面和筛选栏不能为了铺满而变得过高或过长。`game-control.html` 这类配置表单应优先紧凑、少滚动、分组清晰；报表筛选栏里的游戏选择、日期范围、快捷按钮、查询/重置/导出要保持短而顺手，日期框不要拉得很长。
